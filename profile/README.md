## 定位

iwantools 是一个以 AI Coding 为核心工作方式的一人公司，用产品、自动化、标准、playbook 和 skill 叠加个人杠杆。

目标不是多开几个仓库，而是把每天真实发生的开发、决策、复盘和交付，沉淀成一套可复制、可审查、可长期演进的个人公司操作系统。

## 适合合作

| 场景 | 可以一起做什么 |
| --- | --- |
| **AI Coding 落地** | 把真实开发流程整理成标准、playbook、agent skill 和可验证的执行链路 |
| **个人工具 / 小型产品** | 从问题梳理、MVP 设计到前后台实现、发布和复盘 |
| **工程与文档治理** | 梳理仓库边界、技术栈默认值、文档入口、发布检查和长期维护规则 |

## 工作方式

- 先讲清边界、目标和验收标准，再进入实现。
- 小步推进，每次改动都尽量可读、可验证、可回滚。
- 让代码、文档、决策记录和复盘一起沉淀。
- 不追求一次性炫技，优先选择长期可维护的方案。

## 系统版图

| Layer | What it compounds |
| --- | --- |
| **Products** | 个人工具、公开站点、工作台和自动化能力 |
| **Standards** | 工程、文档、Git、安全、发布和技术栈决策 |
| **Playbooks** | 从想法到 PRD、设计、实现、发布、复盘的推进流程 |
| **AI Skills** | 可复用的 Codex/Agent 工作能力 |
| **Operating Board** | 项目登记、路线图、ADR、复盘和下一步行动 |

## 当前建设重点

- 建立稳定的工作区边界，让 AI agent 能用最小上下文做最小可验证改动。
- 把高频工作流沉淀为 standards、playbooks 和 skills，降低重复决策成本。
- 用真实项目反向校准产品方向、工程规则、交付节奏和自动化能力。
